#### Ядро системы
Справочники: сотрудники (ФИО, должность, подразделение), виды вакцин, заболевания/инфекции.   
Учёт вакцинаций: дата прививки, вид вакцины, серия/номер препарата, срок годности ампулы, кто провёл (медработник). Срок годности ампулы обязателен при создании и редактировании записи.  
Контроль сроков: автоматический расчёт даты следующей вакцинации и отображение сотрудников с истекающим/истёкшим сроком.

#### Базовая отчётность
Список сотрудников, которым требуется ревакцинация в ближайший период (неделя/месяц). Статистика по охвату вакцинацией — по подразделениям, по видам вакцин. Индивидуальная карточка сотрудника с историей прививок.

##### Разграничение доступа
Роли:
- Person - пользователь, который видит только себя: свою карточку, свои прививки, свои уведомления.
- HR - пользователь, который видит всех сотрудников своего подразделения, может смотреть отчёты по охвату, но не редактирует медицинские данные.
- Medical worker - пользователь, который вносит и редактирует записи о вакцинациях, работает со справочником вакцин, формирует медицинские отчёты.
- Admin - пользователь, который CRUD по сотрудникам, подразделениям, управление ролями, полный доступ.
